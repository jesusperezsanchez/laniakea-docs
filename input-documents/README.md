# Input Documents

This directory collects community contributions to the Laniakea documentation.

## How to Contribute

1. **Fork this repository** and create a new branch
2. **Add a new markdown file** to this directory with a descriptive name (e.g., `risk-framework-clarification.md`, `new-halo-type-suggestion.md`)
3. **Submit a pull request** for review

## What to Include

Input documents can contain any of the following:

- **Corrections** — Errors or outdated information you've identified
- **New information** — Data, parameters, or context that should be documented
- **Questions** — Clarifications needed about existing documentation
- **Opinions** — Perspectives on design decisions or trade-offs
- **Suggestions** — Proposed improvements or new content areas

## Format

No strict format required. Clear, concise writing is appreciated. If referencing existing documents, include the file path and relevant section.

## What Happens Next

Contributors with write access will:
1. Review submitted input documents
2. Merge relevant content into the appropriate core documentation files
3. Close the PR with a summary of changes made

Input documents may be archived or removed after processing.
